preface.md

前言：我们日常工作中已开始使用多种 Agent 工具（如云端的 ArkClaw，本地桌面端的 QoderWork / Workbuddy 等）。

传统大模型更像“问答助手”，用户提出一个问题，模型给出一段回答；而 Agent 工具可以根据目标，自动拆解任务、规划步骤、调用工具、读取与修改本地文件，并在执行中根据反馈不断修正，交付相对完整的工作成果。

💡 Agent 产品的任务交付质量取决于：模型 + 框架本身。核心价值是让大模型具备更强的任务理解、过程执行和结果交付能力。

Agent 产品的底层逻辑

大模型配合执行框架，实现从任务拆解到成果交付的闭环

长链路任务执行

Agent 工具的执行核心在于对复杂任务的长周期执行和自适应调整：

1. 目标输入 用户提出复杂目标
2. 意图识别 & 规划 大模型自动拆解步骤
3. 工具调用 读取/调用 MCP 或 Skill
4. 反馈修正 & 交付 检查运行结果并修正输出

Coding 方式执行任务

Agent 执行任务的本质是将任务拆分为具体的代码/函数节点（例如“读取 PDF，总结关键点”）。在每个具体节点，通过调用函数、依赖库、Skills 或 MCP，以编程脚本的形式完成控制和处理。

云端 Agent 与 桌面端 Agent 的区别

运行环境的不同决定了它们各自的应用主场

ArkClaw 云端 Agent 平台

核心特点：部署在云端服务器运行。

优势：7×24小时在线、不需要本地电脑保持开机、支持远程触发、跨设备随时查看使用。
定位：适合做“云端长期运行任务助手”。

QoderWork / WorkBuddy 桌面端 Agent 应用

核心特点：作为客户端软件直接在本地电脑运行。

优势：可直接安全接触本地文件及整个办公生态，极佳地适合处理 Excel、文档项目、本地代码库、音视频素材文件等具体的本机工作任务。
定位：适合做“本地办公执行与操作助手”。

前置知识

Qoderwork 两大效率工具：Skill 与 MCP

Skill

专项技能包

通俗理解
给 Agent 提前准备好的“专项工具包/方案包”
解决什么问题
让 Agent 知道遇到某类具体任务（如合并 PDF、解析文档）时，应该采用什么步骤、调用什么依赖、生成什么处理脚本。
💡 通俗案例：把大象装进冰箱
Skill 告诉 Agent：要把大象放进冰箱分为三步（第一步打开门 ➡️ 第二步放大象 ➡️ 第三步关门）。它为任务提供了执行的步骤蓝图。

MCP

连接工具的协议插头

通俗理解
让 Agent 能无缝连接各种外部软件与能力的“插头协议”
解决什么问题
打破大模型“沙盒限制”，让 Agent 可以调用本地文件、外部数据库、浏览器控制、终端命令以及企业钉钉、飞书等生态。
💡 通俗案例：把大象装进冰箱
MCP 给 Agent 接上“机械手”或“遥控器”：Agent 本身只是个大模型，它自己不能去拉冰箱门；有了 MCP，它才能控制外部工具去执行动作，达成放进冰箱的目标。

实践 2：钉钉自定义机器人群发消息

配置自定义机器人 Webhook，调用 MCP 实现定时提醒与 @全体成员

创建并配置钉钉自定义机器人：

  1. 打开钉钉群设置，选择“智能群助手”
  2. 点击“添加机器人”，选择“自定义(Webhook)”
  3. 设置机器人安全设置（如自定义关键词“提醒”或IP限制）
  4. 复制机器人生成的 Webhook 地址

复制机器人的 Webhook，向 Agent 下达以下指令：

请使用mcp，通过这个（https://oapi.dingtalk.com/robot/send?access_token=90d8e737faa0510de4f512d370c4dad9c28e516734f78b177c6c318e8a2d5f74）机器人，在群里发送周报填写提醒：请大家在今天下班前提交周报，并且@全体成员

💡 机器人将调用 Webhook 并利用 MCP 在指定时间在群里发送提醒消息，实现高度自动化的群内跟进。

⚠️ 安全提示：Webhook Access Token 具有群发送权限，请妥善保管不要泄露给外部人员。

实践 4：消息渠道配置（手机端遥控电脑工作）

通过微信/钉钉消息通道，直接在手机上控制电脑端 Agent 执行本地任务

消息通道配置

通过在 QoderWork 配置即时通讯通道，可以将你的个人微信或钉钉变成 Agent 的远程对话入口。

  • 推荐通道：首选微信，扫码即可绑定。
  • 钉钉通道：钉钉配置由于涉及组织限制，一般需要对应的权限申请。
  • 运行限制：此方式下，受控的桌面电脑需保持在开机并处于活跃（未休眠）状态。
📱 移动协同场景：在上班路上或家里，通过微信聊天直接给电脑发一条消息，电脑端的 Agent 就能把表格算好、文案改好，实现极其便捷的无缝移动办公！
* 手机指令发送会通过微信/钉钉中继传输，保证传输通道的安全性。

自定义 OpenAI 兼容模型配置

在 WorkBuddy 中配置 OpenAI 兼容模型，请按以下步骤操作：

  1. 配置文件路径：~/.workbuddy/models.json
  2. 配置 SenseNova 模型：
在 WorkBuddy 中配置 OpenAI 兼容模型，请按以下步骤操作：

1. 配置文件路径
~/.workbuddy/models.json

2.配置 SenseNova 模型
{
  "models": [
    {
      "id": "deepseek-v4-flash", 
      "name": "deepseek-v4-flash",
      "vendor": "SenseNova",
      "apiKey": "你的api_key", 
      "url": "https://token.sensenova.cn/v1/chat/completions",
      "maxInputTokens": 131072,
      "maxOutputTokens": 8192,
      "supportsToolCall": false,
      "supportsImages": false,
      "supportsReasoning": false
    }
  ]
}

💡 模型替换支持：可复制名称 sensenova-6.7-flash-lite、sensenova-u1-fast、deepseek-v4-flash 并在配置文件中替换。
